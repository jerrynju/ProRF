# ProRF 移动端 App 界面设计草案

对应原型文件：[mobile-app-ui-draft.html](mobile-app-ui-draft.html)

## 设计目标

这版草案基于附件 UI 的视觉方向，扩展为一套更完整的移动端 App 主流程。它不是营销页，而是面向 RF 工程师日常使用的工作台界面：快速检查链路、编辑计算图、调整节点参数、管理场景并查看结果。

核心原则：

- 图结构优先：所有页面都围绕 Workflow Graph、Node、Edge、Scenario、Result 展开。
- 数值优先：关键计算结果永远在当前屏幕上方或卡片右侧可见。
- 移动端可操作：复杂画布通过工具条、底部操作区和 Inspector 分层承载。
- 颜色即语义：节点类型颜色在流程、画布、节点库、结果图中保持一致。

## 覆盖页面

1. 工作流流程页
   - 展示链路从 TX 到 RX 的纵向流程。
   - 顶部显示链路增益、系统余量、接收功率。
   - 每张节点卡包含节点类型、关键参数、当前输出值。

2. 图形编辑器
   - 使用移动端画布视图表达 DAG。
   - 顶部工具条负责选择、连接、运行、缩放和自动布局。
   - 底部浮动操作条提供添加节点、添加连接、注释、对子系统。

3. 节点库
   - 左侧竖向分类，右侧节点卡片网格。
   - 节点卡显示图标、名称和最重要的参数 schema。
   - 支持搜索、筛选和后续拖入画布。

4. 节点属性
   - Inspector 作为唯一参数编辑入口。
   - 页内 tab 覆盖参数、输入/输出、结果、图表、诊断。
   - 参数行采用 `value + unit + dimension` 结构，避免裸数值。

5. 结果总览
   - 链路摘要 KPI、瀑布图、瓶颈建议。
   - 适合承接一次执行后的快速审阅。
   - 后续可扩展为导出报告、表格和执行日志。

6. 模板
   - 面向新建工作流的入口。
   - 包含链路预算、卫星链路、雷达方程、OFDM 接收链路、空白工作流。

7. 场景管理
   - 管理参数集和环境差异。
   - 每个场景显示对系统余量的影响。
   - 支持后续扩展为 Sweep、Monte Carlo、版本对比。

8. 诊断中心
   - 聚合单位检查、DAG 检查、压缩区告警、场景风险。
   - 诊断项必须可跳转到节点属性、场景或分析工具。

## 视觉系统

节点颜色：

- 发射机 TX：蓝色 `#367cf6`
- 功率放大器 AMP：橙色 `#ff9f2f`
- 定向耦合器 COUP：青绿色 `#28c59d`
- 天线 ANT：紫色 `#8d5cf6`
- 自由空间传播 PATH：蓝绿色 `#2bb9b2`
- 接收机 RX：橙黄 `#f3a53f`

界面规则：

- 页面背景使用浅灰，内容面板使用白色。
- 卡片圆角控制在 16px 左右，手机外框为展示用途。
- 数值建议在实现中使用等宽字体。
- 正向余量和通过状态使用绿色，风险使用橙色，错误或下降使用红色。
- 不使用大面积渐变背景，不使用装饰性图形。

## 信息架构

底部导航建议：

- 工作流
- 库
- 结果
- 场景
- 设置

工作流内部页签建议：

- 拓扑
- 流程
- 详情
- 版本

Inspector 页签建议：

- 参数
- 输入/输出
- 结果
- 图表
- 诊断

## 后续实现建议

Compose 组件可按以下层次拆分：

- `AppScaffold`：状态栏、顶栏、底部导航。
- `MetricStrip`：顶部三项或多项工程指标。
- `NodeCard`：流程页节点卡。
- `WorkflowCanvas`：移动端画布、节点、连接、minimap。
- `NodeLibraryPanel`：分类和节点库网格。
- `InspectorForm`：schema 驱动参数编辑。
- `ResultSummary`：KPI、瀑布图、瓶颈建议。
- `ScenarioList`：场景卡和对比图。
- `DiagnosticList`：错误、警告、建议。

实现时应保持现有架构约束：UI 只读取 schema、graph、execution result，不直接包含 RF 计算逻辑。
